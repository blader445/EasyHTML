General

Doctype

HTML document type declaration.
Might be at the beginning of every html page.

<!DOCTYPE html>

Paragraphe

Represent a paragraph of text.
Can include image as decoration.

<p>This is a paragraphe.</p>

Html

Represent the parent markup of all documents that use html language.

<html>
  <head>
  </head>
  <body>
  </body>
</html>

Head

Represent the header of the document.

<head>
  <meta charset="UTF-8"/>
  <title>Title of the page</title>
</head>

Meta

Contains the encoding type of the page.
Should be include between head tags.

  <meta charset="UTF-8"/>

Title

Title of the page, shows on the tab in browser.

<title>Title of the page</title>

Elements

Div

Represent a block element in the document.

<div>Ceci est le morceau de code.</div>

Span

Represent an inline element in the document.

<span>Ceci est le morceau de code.</span>

Adress

Represent an adress in the document.
Can be use between article tags or as a single element between body tags.

<address>
  <a href="mailto:jim@rock.com">jim@rock.com</a>
  <a href="tel:+13115552368">(311) 555-2368</a>
</address>

Article

Represent an independant element in the document.

<article>This is an.</article>

Blockquote

Represent a quote block in the document.
You can add author name and source name.

<blockquote cite="https://www.huxley.net/bnw/four.html">
  <p>Words can be like X-rays, if you use them properly—they’ll go through anything. You read and you’re pierced.</p>
  <footer>
    —Aldous Huxley, <cite>Brave New World</cite>
  </footer>
</blockquote>

Br

Represent a line jump in the document.

<br>

Links

Anchor

Represent an anchor that can be use to navigate to a specific place in the document, download something or link to another website.

<a>This is an anchor.</a>

Abbreviation

Represent an important part of the document.

<abbr>This is an abbreviation.</abbr>

E-mail

Represent a link that opens e-mail client.

<a href="mailto:test@gmail.com">Click here to send an e-mail.</a>

Images & medias

Audio

Represent an audio element in the document.
Supported formats are mp3, acc, ogg and wav (to avoid).

<audio controls autoplay loop preload="auto|metadata|none">
  <source src="example.mp3" type="audio/mpeg">
  <source src="example.ogg" type="audio/ogg">
  <source src="example.acc" type="audio/acc">
  <source src="example.wav type="audio/wav">
</audio>

Video

Represent a video in the document.
Supported container formats are AVI, MP4, MKV...
Video supported codecs are H.264, Ogg Theora and WebM.
Audio supported codecs are the same than on audio element.

<video src="test.mp4" poster="image.jpg" width="720px" heigth="480px" controls autoplay loop>
<video/>

Image

Represent an image in the document.

<img src="test.jpg" alt="alternative text"/>

Image in figure

Represent an image contained into a figure on the document.
Should be use with an image directly representing the page, not for decoration.

<figure>
  <img src="test.jpg" alt="alternative text"/>
  <figcaption>This is the figcaption.</figcaption>
</figure>
